## Component Inventory

All new components are vanilla HTML + inline styles. Match existing patterns exactly.

### 1. Modal Overlay (`#edit-modal-overlay`)

```
position: fixed
top: 0; left: 0; width: 100%; height: 100%
background: rgba(0,0,0,0.5)
z-index: 1000
display: flex; align-items: center; justify-content: center
```

- Clicking the overlay backdrop (outside the dialog) calls `closeEditModal()`.
- Pressing Escape calls `closeEditModal()`.

### 2. Modal Dialog (`.em-dialog`)

```
background: #fff
border-radius: 8px
padding: 32px
width: min(560px, 92vw)
max-height: 90vh
overflow-y: auto
position: relative
border: 1px solid var(--border-subtle)
box-shadow: 0 20px 60px rgba(0,0,0,.2)
```

- Matches the visual weight of `.login-box` (white card, rounded, shadow).
- Width capped at 560px to fit all fields without scrolling on desktop.
- `overflow-y: auto` on mobile where 90vh may be reached.

### 3. Modal Header

```
display: flex; justify-content: space-between; align-items: flex-start
margin-bottom: 24px
```

- Left: `<h2>` "EDIT PRODUCT" — 16px, weight 700, letter-spacing 1px, text-transform uppercase, color var(--maroon). Matches `.product-form h2`. **Primary focal point** of the modal at open — highest-contrast text element, orients the owner to context.
- Right: Close button (`×`) — `background:none; border:none; font-size:24px; color:#888; cursor:pointer; line-height:1`. Hover: `color:var(--charcoal)`. Calls `closeEditModal()`. Must include `aria-label="Close edit modal"` for accessibility.

### 4. Form Fields (reuse existing CSS classes)

All fields inside the modal use `.pf-label` and `.pf-input` class names exactly.

| Field | Element | ID | Notes |
|-------|---------|-----|-------|
| Product Name | `<input type="text">` | `em-name` | Pre-filled with `p.name` via `esc()` |
| Price (₹ INR) | `<input type="number" min="1">` | `em-price` | Pre-filled with `p.price` (integer rupees) |

Layout: single-column (not `.form-grid` 2-column). The modal is narrow; 2-column would squeeze inputs. Name and Price stack vertically.

### 5. Size Pills (`em-` prefix on all IDs)

**Toggle function for modal:** `emToggleStock(cb)` — mirrors `toggleStock(cb)` at line 422, but references `em-stock-${cb.value}` instead of `stock-${cb.value}`. Must also update pill background/border colors identically.

**Element IDs for the 7 sizes:**

### 8. Product Tile (click trigger — D-01)

`renderProducts()` must add two attributes to the `.product-tile` `<div>`:

The DELETE button's `onclick` must call `event.stopPropagation()` to prevent the tile click from firing when the user clicks DELETE.

## Interaction Contracts

### IC-01: Tile Click Opens Modal

- Trigger: `onclick` on `.product-tile` div
- Guard: `event.stopPropagation()` on DELETE button prevents cascading
- Effect: `openEditModal(productId)` appends `#edit-modal-overlay` to `document.body`
- Visual feedback: Overlay appears immediately with loading placeholder; form replaces it on API success

### IC-02: Escape Key Closes Modal

- Listen on `document.addEventListener('keydown', ...)` inside `openEditModal`
- Remove listener on `closeEditModal()` (store handler reference to remove it)
- Effect: same as clicking Cancel — `closeEditModal()` removes overlay from DOM

### IC-03: Backdrop Click Closes Modal

- `onclick` on `#edit-modal-overlay` itself (not `.em-dialog`)
- Stop propagation on `.em-dialog` so inner clicks don't bubble to overlay
- Effect: `closeEditModal()`

### IC-04: Save Flow

1. User clicks SAVE CHANGES
2. Collect: `em-name`, `em-price`, `em-description` values
3. Collect checked sizes: `document.querySelectorAll('input[name="em-sizes[]"]:checked')`
4. Validate: at least one size checked — if not, show `#em-sizes-error`, return
5. Call: `apiFetch('/api/products/' + productId, 'PATCH', { name, price, description, sizes, stock })`
6. On success: `closeEditModal()` then `loadProducts()`
7. On error: `alert('Failed to save: ' + e.message)` — modal stays open

### IC-05: Checked Pill Toggle (`emToggleStock`)

Same visual behavior as `toggleStock(cb)` at line 422:
- On check: show stock input (`display:inline-block`), set pill `border-color:var(--maroon)`, `background:var(--cream-warm)`
- On uncheck: hide stock input (`display:none`), reset `value='0'`, restore `border-color:var(--border-subtle)`, `background:#f7f4f0`

### IC-06: Zero-Stock Size Pre-checked (D-07)

When rendering pills in `openEditModal()`:
- If `existingSizes.has(sz)` → checkbox is `checked`, stock input is shown (even if `stockMap[sz] === 0`)
- The owner must explicitly uncheck to remove a zero-stock variant

### IC-07: Modal Cleanup

`closeEditModal()` removes `#edit-modal-overlay` from `document.body` entirely via `element.remove()`. Does not use `display:none` (avoids DOM accumulation on repeated opens).

## Implementation Notes for Executor

1. **ID namespace:** Every element introduced by the modal uses the `em-` prefix.
   Static Add Product form uses `p-name`, `stock-XS`, `size-XS`. Modal uses `em-name`,
   `em-stock-XS`, `em-size-XS`. Both forms coexist in the DOM simultaneously.

2. **XSS:** Every product data value interpolated into modal `innerHTML` must be wrapped
   in `esc()` (line 534). Includes: `p.name`, `p.price`, `p.description`, `sz` (size label).

3. **Toggle function:** Name the modal toggle `emToggleStock(cb)` (not `toggleStock`).
   Inside it reference `document.getElementById('em-stock-' + cb.value)`.

4. **apiFetch for PATCH:** Pass body as plain JS object — `apiFetch` serializes to JSON
   and sets `Content-Type: application/json` automatically (line 528).

5. **Modal z-index:** `z-index: 1000` on the overlay. No existing elements in the
   dashboard establish a stacking context above this value.

6. **No new CSS classes for modal internals:** Use inline styles on modal-specific
   elements. Reuse `.pf-label`, `.pf-input`, `.btn-save`, `.size-row` class names where
   they apply — these classes are already defined and produce the correct visual.

7. **Tile cursor:** Add `cursor:pointer` directly in the `renderProducts()` template
   string on `.product-tile` div. No new CSS rule needed.

8. **Event stop on DELETE:** Inside `renderProducts()`, the DELETE `<button>` `onclick`
   must become `onclick="event.stopPropagation();deleteProduct('${esc(String(p.id))}','${esc(p.name)}')"`

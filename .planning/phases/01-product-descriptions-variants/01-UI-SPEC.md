## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Tailwind class | Usage in this phase |
|-------|-------|---------------|---------------------|
| xs | 4px | `gap-1` | Icon gaps; size-row checkbox-to-label gap |
| sm | 8px | `gap-2` / `p-2` | Checkbox row internal gap |
| md | 16px | `gap-4` / `p-4` | Size button flex gap; form field margin-bottom |
| lg | 24px | `gap-6` / `p-6` | Form section vertical padding |
| xl | 32px | `p-8` | Dashboard form card padding (matches existing `.product-form` 32px) |
| 2xl | 48px | `py-12` | Page-level container padding (existing, do not change) |

Exceptions:
- `12px` — used for `gap: 12px` in `#sizes-container` and `padding: 8px 12px` in `.size-row` pills and textarea. Natural midpoint between `sm` (8px) and `md` (16px) for compact dashboard pill padding. Valid multiple of 4.
- Size buttons are `w-14 h-14` (56px square) — departs from the named scale tokens but 56 is a valid multiple of 4 (56 ÷ 4 = 14). Matches existing button dimensions in `ProductDetailClient.tsx` line 107.
- Dashboard stock input is `width: 72px` — fixed width for 3-digit stock numbers, not a spacing unit. 72 ÷ 4 = 18 (multiple of 4).
- Touch target minimum for any interactive element is 44px height. All size buttons (56px) and dashboard checkboxes (with their labels) satisfy this.

## Component Contracts

**Primary visual anchor:** The full-width maroon CTA button (`bg-poshakh-maroon`, `shadow-lg`) at the base of the product detail page. All other elements (size selector, description accordion) are secondary. This phase adds to an existing page — the existing page hierarchy is preserved, not redesigned.

### 1. Size Selector — `ProductDetailClient.tsx` (frontend)

Replaces hardcoded `['XS', 'S', 'M', 'L', 'XL']` array at line 103 with `product.variants` iteration.

#### Container

```
<div class="flex gap-4">
  {product.variants?.map(v => <SizeButton key={v.size} variant={v} />)}
</div>
```

Layout: `flex` row with `gap-4` (16px). Wraps naturally if 7 sizes exceed container width on mobile.

#### Size button — in-stock, unselected (default state)

```
className="w-14 h-14 border flex items-center justify-center font-bold transition-all
           border-poshakh-gold/50 text-poshakh-charcoal
           hover:border-poshakh-maroon hover:text-poshakh-maroon"
```

#### Size button — in-stock, selected (active state)

```
className="w-14 h-14 border flex items-center justify-center font-bold transition-all
           border-poshakh-maroon bg-poshakh-maroon text-poshakh-gold"
```

#### Size button — out of stock (OOS state, stock === 0)

```
disabled={true}
className="w-14 h-14 border flex items-center justify-center font-bold transition-all
           opacity-50 line-through cursor-not-allowed
           border-poshakh-gold/30 text-poshakh-charcoal/40"
```

Behaviour:
- `disabled` attribute set to `true` — prevents click events at the DOM level
- `line-through` via Tailwind `line-through` class applies `text-decoration: line-through`
- `opacity-50` reduces both border and text to 50% opacity (consistent with `globals.css` `button:disabled { opacity: 0.55 }` — we override to 50% explicitly via class for the finer OOS aesthetic)
- Click handler guard: `onClick={() => !oos && setSelectedSize(v.size)}` — redundant with `disabled` but defensive

#### Pre-selection logic

```typescript
const firstInStock = product.variants?.find(v => v.stock > 0)?.size ?? null;
const [selectedSize, setSelectedSize] = useState<string | null>(firstInStock);
```

If `firstInStock` is `null` (all OOS or no variants): `selectedSize` is `null`. No size button renders as selected.

#### All-OOS detection

```typescript
const allOOS = !product.variants?.length || product.variants.every(v => v.stock === 0);
```

Used to swap CTA. Also covers the D-13 case (existing products with no variants).

### 2. Add to Cart CTA — `ProductDetailClient.tsx` (frontend)

Conditional render — mutually exclusive states:

#### Normal state (at least one variant in stock)

```
<button
  onClick={handleAddToCart}
  className="w-full py-5 bg-poshakh-maroon text-poshakh-gold font-heading tracking-widest text-lg uppercase font-bold hover:bg-poshakh-maroon-dark transition-colors shadow-lg"
>
  Add to Cart
</button>
```

#### Out of Stock state (allOOS === true)

```
<button
  disabled
  className="w-full py-5 bg-poshakh-charcoal/30 text-white font-heading tracking-widest text-lg uppercase font-bold cursor-not-allowed"
>
  Out of Stock
</button>
```

- Background: `bg-poshakh-charcoal/30` — 30% opacity charcoal. Visually muted, clearly non-interactive.
- Text: `text-white` — readable on the 30%-opacity dark background.
- No hover state — `disabled` suppresses interactions; `cursor-not-allowed` communicates non-interactivity.
- No `shadow-lg` — disabled buttons do not have elevation.

#### Sticky bar CTA (scroll-triggered, bottom of viewport)

The sticky bar CTA at the bottom of the page (appears after 600px scroll) also uses `handleAddToCart`. Apply the same OOS swap logic there:

Normal: `bg-poshakh-maroon text-poshakh-gold font-heading tracking-widest uppercase font-bold px-8 py-3 hover:bg-poshakh-maroon-dark transition-colors`

OOS: `bg-poshakh-charcoal/30 text-white font-heading tracking-widest uppercase font-bold px-8 py-3 cursor-not-allowed` + `disabled`

### 4. Dashboard — Description Textarea (backend/dashboard/index.html)

Insert after the Collection field in `.form-grid`, spanning full width (`.full` class):

#### HTML structure

```html
<div class="full">
  <label class="pf-label" for="p-description">Description (optional)</label>
  <textarea
    class="pf-input"
    id="p-description"
    rows="4"
    placeholder="Describe the product — fabric, occasion, care instructions..."
    style="height:auto;resize:vertical;padding:8px 12px;line-height:1.5"
  ></textarea>
</div>
```

Styling inherits `.pf-input` — `border: 1px solid #ddd`, `font-size: 14px`, `border-radius: 4px`, focus state `border-color: var(--maroon)` (from global focus rule in `globals.css`). The `resize: vertical` allows the owner to expand the textarea; `rows="4"` gives 4 lines of initial height (~88px). `padding: 8px 12px` instead of the input's fixed `height: 44px` since textarea height is content-driven (8px is the `sm` token in the spacing scale).

Reset after successful save: `document.getElementById('p-description').value = '';`

### 5. Dashboard — Size Checkboxes + Stock Inputs (backend/dashboard/index.html)

Insert after the Description field, spanning full width:

#### HTML structure

```html
<div class="full">
  <label class="pf-label">Sizes & Stock <span style="color:var(--error)">*</span></label>
  <div id="sizes-container" style="display:flex;flex-wrap:wrap;gap:12px;margin-top:8px">
    <!-- Repeated for each size: XS, S, M, L, XL, XXL, Free Size -->
    <div class="size-row" style="display:flex;align-items:center;gap:8px;background:#f7f4f0;border:1px solid var(--border-subtle);border-radius:4px;padding:8px 12px">
      <input type="checkbox" id="size-XS" name="sizes[]" value="XS" onchange="toggleStock(this)" style="width:16px;height:16px;accent-color:var(--maroon);cursor:pointer"/>
      <label for="size-XS" style="font-size:12px;font-weight:700;letter-spacing:1px;cursor:pointer;min-width:36px">XS</label>
      <input type="number" id="stock-XS" name="stock[]" min="0" value="0" placeholder="qty"
             style="display:none;width:72px;height:32px;border:1px solid #ddd;border-radius:4px;padding:0 8px;font-size:12px;text-align:center"/>
    </div>
    <!-- ...repeat for S, M, L, XL, XXL, Free Size -->
  </div>
  <p id="sizes-error" style="color:var(--error);font-size:12px;margin-top:8px;display:none">Select at least one size.</p>
</div>
```

#### Size pool and order

`XS` | `S` | `M` | `L` | `XL` | `XXL` | `Free Size` — exactly this order, left-to-right, wrapping to next row on narrow viewports.

#### Checkbox pill visual states

| State | Border | Background |
|-------|--------|------------|
| Unchecked | `1px solid var(--border-subtle)` (#e5e7eb) | `#f7f4f0` (= `var(--bg)`) |
| Checked | `1px solid var(--maroon)` (#3D0D16) | `var(--cream-warm)` (#fdf8f3) |

The `accent-color: var(--maroon)` CSS property colours the native checkbox tick. The pill border change (unchecked → checked) is applied via JS `toggleStock()`:

```javascript
function toggleStock(cb) {
  const row = cb.closest('.size-row');
  const stockInput = document.getElementById('stock-' + cb.value);
  if (cb.checked) {
    stockInput.style.display = 'inline-block';
    row.style.borderColor = 'var(--maroon)';
    row.style.background = 'var(--cream-warm)';
  } else {
    stockInput.style.display = 'none';
    stockInput.value = '0';
    row.style.borderColor = 'var(--border-subtle)';
    row.style.background = '#f7f4f0';
  }
}
```

#### Stock number input (visible only when checkbox is checked)

- Width: `72px` — fits 3-digit quantities comfortably
- Height: `32px` — compact; the pill is `padding: 8px 12px` so total pill height is ~48px with the input
- `text-align: center` — stock numbers look cleaner centred
- `min="0"` — negative stock disallowed at input level
- Placeholder: `qty`

#### Validation indicator

- Required asterisk `*` in `color: #c00` next to the "Sizes & Stock" label
- Error paragraph `#sizes-error` (hidden by default) appears below the container when the owner clicks Save with no sizes checked
- Error text: `Select at least one size.` (see Copywriting Contract)
- Error clears on next `saveProduct()` call that passes validation

#### Reset after save

```javascript
document.querySelectorAll('input[name="sizes[]"]').forEach(cb => {
  cb.checked = false;
  toggleStock(cb);
});
```

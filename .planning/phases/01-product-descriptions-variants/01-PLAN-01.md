In `backend/db.js`, inside the `initDb()` function, add the following block immediately after the closing `);` of the existing `CREATE TABLE IF NOT EXISTS products` DDL block and before the `CREATE TABLE IF NOT EXISTS orders` block:

```javascript
  // Phase 01: add description column — idempotent via try/catch
  try {
    _db.run(`ALTER TABLE products ADD COLUMN description TEXT`);
  } catch (_) {
    // Column already exists — safe to ignore on subsequent startups
  }
```

Rules for this change:
- Use `_db.run()` directly (not `db.prepare()`) — we are inside `initDb()` before the `db` wrapper is used. This matches the existing DDL pattern in the file.
- Do NOT add `NOT NULL` to the description column — SQLite ALTER TABLE ADD COLUMN rejects NOT NULL without a DEFAULT. `TEXT` (nullable) is correct per D-12.
- The try/catch is mandatory — SQLite throws if the column already exists. Without it the server crashes on second startup.
- Do NOT add a `DEFAULT ''` to description — NULL is the correct state for existing products (D-12), and `formatProduct()` already coerces null to `""`.

In `backend/db.js`, inside `initDb()`, add the following DDL block immediately after the `try/catch` block for the `description` ALTER TABLE (added in Task 1), and before the `CREATE TABLE IF NOT EXISTS orders` block:

```javascript
  // Phase 01: product variants table — per D-01 / D-02
  _db.run(`
    CREATE TABLE IF NOT EXISTS product_variants (
      id         INTEGER PRIMARY KEY AUTOINCREMENT,
      product_id INTEGER NOT NULL REFERENCES products(id) ON DELETE CASCADE,
      size       TEXT    NOT NULL,
      stock      INTEGER NOT NULL DEFAULT 0,
      UNIQUE(product_id, size)
    )
  `);
```

Rules for this change:
- `CREATE TABLE IF NOT EXISTS` is idempotent — no try/catch needed.
- The FK constraint uses `REFERENCES products(id) ON DELETE CASCADE` — not a separate `FOREIGN KEY(...)` clause. Both forms are valid in SQLite; inline form is consistent with the simple schema style in this file.
- `PRAGMA foreign_keys = ON` is already set at line 28 — the CASCADE will work correctly.
- Column order: id, product_id, size, stock — per D-02. The UNIQUE constraint is a table-level constraint at the end.
- `stock INTEGER NOT NULL DEFAULT 0` — per D-02. NOT NULL with DEFAULT is allowed by SQLite ALTER TABLE rules (but this is a CREATE TABLE, not ALTER, so no restriction applies).
- Do NOT use `INTEGER` for `product_id` as a FOREIGN KEY column with a named constraint — the inline `REFERENCES` syntax is sufficient and matches SQLite documentation.
  </action>

# Plan 04-03: Live Deployment & Smoke Test

**Phase:** 04-production-deploy
**Objective:** Execute the deployment and verify the live system.

## Context
Final step to get Poshakh live. Requires manual steps for Railway and Vercel dashboard configuration.

## Tasks

### 1. Railway Infrastructure Setup
- **Action:** 
    - Create a new project on Railway.
    - Connect the GitHub repository.
    - Add a **Volume** of 1GB and mount it at `/app/data`.
    - Configure the environment variables identified in 04-PLAN-01.
- **Verification:** Backend service status is "Success".

### 2. Vercel Frontend Deployment
- **Action:**
    - Import the project into Vercel.
    - Set root directory to `frontend`.
    - Configure environment variables:
        - `NEXT_PUBLIC_BACKEND_URL`: [Your Railway Backend URL]
        - `COOKIE_SECRET`: [Random String]
        - `COOKIE_SECURE`: `true`
- **Verification:** Vercel deployment URL is accessible.

### 3. Verification & Smoke Test
- **Action:**
    - Navigate to the live Vercel URL.
    - **Test 1:** Log in as owner on the live backend (if dashboard is public) or via API.
    - **Test 2 (Customer):** Add a product to cart -> Checkout -> Fill info -> Choose UPI -> Enter dummy UTR -> Confirm.
    - **Test 3 (Owner):** Log in to dashboard and verify the new order appears with status `pending_verification`.
- **Verification:** Order successfully persisted in the live database.

## Verification
- App is fully functional on live subdomains.
- Images are loading from the live `/uploads` endpoint.

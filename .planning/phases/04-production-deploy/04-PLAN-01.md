# Plan 04-01: Backend Production Prep

**Phase:** 04-production-deploy
**Objective:** Configure the Express backend for the Railway environment.

## Context
Downstream from [04-CONTEXT.md](04-CONTEXT.md). We need to ensure the backend is ready for a containerized environment with persistent volumes and secure cookie handling.

## Tasks

### 1. Trust Proxy Configuration
- **File:** `backend/server.js`
- **Action:** Add `app.set('trust proxy', 1)` to the Express app. This is required for Express to correctly identify HTTPS connections forwarded by the Railway/Vercel load balancers, ensuring secure cookies are sent.
- **Verification:** Code review.

### 2. Database Path Verification
- **File:** `backend/db.js`
- **Action:** Ensure the `data/` directory is used for the SQLite file.
- **Verification:** Run `node backend/db.js` and verify it creates/connects to `data/poshakh.db`.

### 3. Environment Variable Documentation
- **File:** `backend/PROD_ENV_CHECKLIST.md` (New)
- **Action:** Create a checklist of all mandatory environment variables for Railway:
    - `OWNER_EMAIL`
    - `OWNER_PASSWORD`
    - `JWT_SECRET`
    - `STORE_CORS` (set to Vercel URL)
    - `PORT` (Railway provides this)
- **Verification:** File exists.

## Verification
- Run `npm run build` (if any) in backend to ensure no compilation errors.
- Start server locally with `NODE_ENV=production` and check for any startup errors.

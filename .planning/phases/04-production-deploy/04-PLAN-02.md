# Plan 04-02: Frontend Production Prep

**Phase:** 04-production-deploy
**Objective:** Update the Next.js frontend for Vercel and disable Razorpay.

## Context
Downstream from [04-CONTEXT.md](04-CONTEXT.md). We are disabling Razorpay for the v1 launch and switching to manual UPI/COD only.

## Tasks

### 1. Disable Razorpay UI
- **File:** `frontend/src/app/checkout/page.tsx`
- **Action:** 
    - Comment out or remove the "PAY WITH RAZORPAY" button (lines 407-413).
    - Ensure the "PAY WITH UPI" button (lines 416-422) is the primary option.
    - Set `upiStep` to "idle" by default (already done).
- **Verification:** Visual check in browser.

### 2. Configure Production API URL
- **File:** `frontend/src/app/checkout/page.tsx` (and any other files using `BACKEND`)
- **Action:** Ensure `process.env.NEXT_PUBLIC_BACKEND_URL` is used as the base URL for all API calls.
- **Verification:** `grep -r "http://localhost:9000" frontend/src` should return no hits in production-critical paths.

### 3. Enforce Secure Cookies
- **File:** `frontend/src/app/api/auth/login/route.ts` (and logout/signup)
- **Action:** When setting cookies, set `secure: process.env.NODE_ENV === 'production'`.
- **Verification:** Check `cookie` options in auth API routes.

## Verification
- Run `npm run build` in the `frontend` directory.
- Verify that the build completes successfully and Razorpay scripts are not being called unnecessarily.
